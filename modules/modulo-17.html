Nomes de Arquivos

Nomes consistentes e descritivos facilitam a navegacção e manutenção do projeto.

✅ Boas práticas:

  • Use kebab-case: minha-pagina.html
  • Seja descritivo: lista-produtos.html
  • Evite espaços e caracteres especiais
  • Use apenas letras, números e hífens

❌ Ruim:

Minha Página.html
file1.html
pagina_do_site.html

✅ Bom:

minha-pagina.html
lista-produtos.html
sobre-empresa.html
📄 contacto.html
📄 galeria-imagens.html
📄 Página Inicial.html

Estrutura de Pastas

Uma estrutura organizada facilita encontrar arquivos e escalar o projeto.

📁 Estrutura recomendada:

projeto/
├── index.html
├── css/
│   ├── styles.css
│   └── module.css
├── js/
│   ├── main.js
│   └── module.js
├── images/
│   ├── logo.png
│   └── favicon.ico
└── modules/
    └── modulo-1.html

🎯 Princípios:

  • Uma pasta por tipo de arquivo
  • Nomes em minúsculas
  • Sem espaços em nomes de pastas
  • Consistência em todo o projeto

Versionamento Semântico

Versionamento semântico (SemVer) comunica mudanças de forma clara: MAJOR.MINOR.PATCH.

📈 Regras do SemVer:

  • MAJOR: Mudanças incompatíveis
  • MINOR: Novas funcionalidades compatíveis
  • PATCH: Correções de bugs

🔖 Exemplos:

1.0.0 - Primeira versão estável
1.1.0 - Adicionada funcionalidade X
1.1.1 - Corrigido bug no formulário
2.0.0 - Mudança na API (incompatível)
1.0.0 Lançamento
1.1.0 Funcionalidade
2.0.0 Quebra
