Aula 8: Serviços e Injeção de Dependência

1. Objetivo da Aula

Entender o que são Serviços (Services) em Angular e como eles são usados para compartilhar lógica de negócio e dados entre componentes através da Injeção de Dependência.

2. Teoria com Analogia

Imagine que seus componentes são diferentes cômodos de uma casa (sala, quarto, cozinha). Agora, imagine que você precisa de acesso à internet em todos os cômodos. Em vez de cada cômodo ter seu próprio modem e roteador (lógica duplicada), você instala um único roteador potente (um Serviço) em um local central. Qualquer cômodo que precise de internet (dados ou uma função) simplesmente se conecta a este serviço central. Isso é o que um Serviço faz: centraliza uma funcionalidade para que múltiplos componentes possam usá-la sem duplicar código. A "Injeção de Dependência" é o ato de "conectar" o cômodo ao roteador.

3. Prática Apoiada

Vamos ver como criar um serviço para fornecer uma lista de usuários e injetá-lo em um componente.

Passo 1: Criar o Serviço (`user.service.ts`)


import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class UserService {
  getUsers() {
    return ['Alice', 'Bob', 'Charlie'];
  }
}

Passo 2: Injetar e Usar no Componente (`user-list.component.ts`)


import { Component, OnInit } from '@angular/core';
import { UserService } from '../user.service';

@Component({
  selector: 'app-user-list',
  template: `
    <ul>
      <li *ngFor="let user of users">{{ user }}</li>
    </ul>
  `
})
export class UserListComponent implements OnInit {
  users: string[];

  constructor(private userService: UserService) { }

  ngOnInit() {
    this.users = this.userService.getUsers();
  }
}

4. Perguntas de Múltipla Escolha

Para que serve o decorator `@Injectable()`?

a) Para criar um componente.
b) Para marcar uma classe como disponível para ser injetada como uma dependência.
c) Para definir o template HTML de um serviço.

5. Resumo do que Aprendi

Nesta aula, aprendemos que Serviços em Angular são usados para organizar e compartilhar código. Eles são injetados nos componentes, seguindo o padrão de Injeção de Dependência, o que torna nossa aplicação mais modular e fácil de manter.
