Objetivo da Aula

O objetivo é integrar todo o conhecimento do curso de Angular para construir uma Single Page Application (SPA) funcional, como uma lista de tarefas (To-Do List), utilizando componentes, serviços e roteamento.

Teoria com Analogia

Pense em construir uma aplicação Angular como montar um carro complexo com peças de LEGO. Cada peça é um componente (roda, porta, motor). Os serviços são como o sistema elétrico que fornece energia para todos. O roteamento é o GPS que te leva para diferentes partes do carro (diferentes telas).

Prática Apoiada

O desafio é criar uma aplicação de To-Do List. Crie um componente para exibir a lista de tarefas, um componente para adicionar novas tarefas e um serviço para gerenciar os dados. Use o roteamento para navegar entre uma página "Home" e a página da "Lista de Tarefas".


// Exemplo de um serviço de tarefas
import { Injectable } from '@angular/core';

@Injectable({ providedIn: 'root' })
export class TaskService {
  private tasks = ['Estudar Angular', 'Fazer o projeto final'];

  getTasks() { return this.tasks; }
  addTask(task: string) { this.tasks.push(task); }
}

Perguntas de Múltipla Escolha

Qual é a principal responsabilidade de um Serviço em Angular?

A. Controlar a interface do usuário e a lógica de exibição.
B. Compartilhar dados e lógica de negócios entre componentes.
C. Definir os estilos da aplicação.

Explicação: Serviços são usados para encapsular lógica que não está diretamente ligada à visão, como busca de dados ou comunicação com APIs. Eles são injetáveis e ajudam a manter os componentes mais limpos e focados na UI.
